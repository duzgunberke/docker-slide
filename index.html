Docker'ın Tanımı ve Amacı


Docker, 2013 yılında başlatılan ve konteynerleştirme teknolojisini temel alan bir açık kaynak projesidir.

Konteynerleştirme, uygulamaların, bağımlılıklarının ve yapılandırmalarının izole bir şekilde çalıştırılabilmesini sağlayan bir sanallaştırma yöntemidir. Docker, bu konteynerleştirme teknolojisini kullanarak yazılımları birbirinden bağımsız ve taşınabilir bir şekilde çalıştırmanızı sağlar.

Docker'ın ana amacı, uygulamaların geliştirme, dağıtım ve çalıştırma süreçlerini kolaylaştırmaktır.

Geleneksel olarak, bir uygulama bir sunucuda doğrudan çalıştırılırken, Docker konteynerleri, uygulama ve bağımlılıklarını birleştiren taşınabilir birimlerdir. Bu konteynerler, işletim sistemi seviyesinde izolasyon sağlayarak, uygulamaların farklı ortamlarda tutarlı bir şekilde çalışmasını sağlar.

Konteyner Teknolojisinin Önemi


Konteynerler, uygulamaların ve bağımlılıklarının herhangi bir ortamda tutarlı bir şekilde çalışmasını sağlar ve böylece geliştirme, test ve dağıtım süreçlerini hızlandırır.
Konteynerler ve sanal makineler arasındaki farklar: Konteynerler, daha hafif ve daha hız başlatılabilen izole ortamlar sağlarken, sanal makineler daha ağır ve daha fazla kaynak tüketir.

Docker'ın popülaritesi ve kullanım alanları

Docker, özellikle mikro hizmetler ve DevOps uygulamalarında yaygın olarak kullanılır ve geliştiricilerin uygulamalarını daha hızlı ve güvenli bir şekilde dağıtmalarına olanak tanır.

Daha yakından Docker'ın temel bileşenleri

Docker Engine

  • Docker'ın temel bileşeni olan Docker Engine, Docker API'si ve Docker Daemon'u içerir.
  • Docker Daemon, Docker API ile iletişim kurarak konteynerlerin yönetilmesini ve çalıştırılmasını sağlar

Docker Engine

Docker Image

  • Docker Image'lar, uygulamaların ve bağımlılıklarının paketlendiği ve dağıtıldığı okunabilir ve yazılabilir dosya sistemleridir.
  • Docker Image katmanları: Docker Image'lar, katmanlı bir yapıya sahiptir ve her katman, önceki katman üzerinde yapılan değişiklikleri temsil eder. Bu, Image boyutunu azaltır ve güncellemi hızlandırır

Docker Image

Docker Container

  • Docker Container'lar, Docker Image'lardan çalıştırılan ve uygulamaların çalıştığı izole edilmiş ortamlardır.
  • Docker Container'lar, çalışan, durdurulan ve duraklatılan gibi farklı durumlar alabilir.

Docker Hub ve Registry

  • Docker Hub, Docker Image'ların paylaşıldığı ve saklandığı merkezi bir depodur.
  • Docker Registry ise özel Docker Image'larınızı saklayabileceğiniz ve yönetebileceğiniz bir depodur.

Docker File ASP.NET Core


						FROM mcr.microsoft.com/dotnet/sdk:6.0 AS build-env
						WORKDIR /app
						EXPOSE 8080

						# Copy csproj and restore as distinct layers
						COPY "MakeEvents.sln" "MakeEvents.sln"
						COPY "API/API.csproj" "API/API.csproj"
						COPY "Domain/Domain.csproj" "Domain/Domain.csproj"
						COPY "Infrastructure/Infrastructure.csproj" "Infrastructure/Infrastructure.csproj"
						COPY "Application/Application.csproj" "Application/Application.csproj"
						COPY "Persistence/Persistence.csproj" "Persistence/Persistence.csproj"

						RUN dotnet restore "MakeEvents.sln"

						# Copy everything else and build
						COPY . .
						WORKDIR /app
						RUN dotnet publish -c Release -o out

						# Build runtime image
						FROM mcr.microsoft.com/dotnet/aspnet:6.0
						WORKDIR /app
						COPY --from=build-env /app/out .
						ENTRYPOINT ["dotnet", "API.dll"]

Docker File Flask


						# Temel alınacak image belirtin (bu örnekte Ubuntu 20.04)
						FROM ubuntu:20.04

						# İmajın yazarını belirtin
						LABEL maintainer="yourname@example.com"

						# Çalışma dizinini belirtin (uygulamanızın dosyalarını buraya kopyalayacaksınız)
						WORKDIR /app

						# Gerekli paketleri güncelleyin ve yükleyin
						RUN apt-get update && \
							apt-get upgrade -y && \
							apt-get install -y python3 python3-pip

						# Uygulamanızın gereksinimlerini yükleyin (requirements.txt dosyasını kullanarak)
						COPY requirements.txt .
						RUN pip3 install -r requirements.txt

						# Uygulamanızın kaynak kodunu kopyalayın
						COPY src/ /app

						# Uygulamanın çalıştırılması için gerekli olan ortam değişkenlerini belirtin
						ENV FLASK_APP=app.py
						ENV FLASK_RUN_HOST=0.0.0.0

						# Uygulamanın çalıştırılması için gerekli olan bağlantı noktasını belirtin
						EXPOSE 5000

						# Uygulamanın nasıl çalıştırılacağını belirtin
						CMD ["flask", "run"]
